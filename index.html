    RobotCode2026Public Documentation

    This site documents 6328's 2026 robot code for Darwin. It is written for students, mentors, and contributors who need to understand how the code is organized, how to run it safely, and how new features should fit into the existing architecture.

    The repository uses WPILib command-based programming, AdvantageKit logging and replay, generated Choreo trajectories, Northstar vision, and a strict hardware abstraction pattern. The most important idea is that subsystem logic should be deterministic and replayable: hardware data enters through IO input objects, robot logic acts on those inputs, and outputs are applied after the scheduler runs.

    Current Implementation Status

    This public branch contains the complete architecture and many subsystem implementations, but some real hardware IO classes are intentionally not present or not wired yet. RobotContainer currently creates SIMBOT implementations for physics simulation and falls back to no-op IO implementations for replay or unimplemented robot hardware paths. When adding real hardware support, keep subsystem logic unchanged and add the hardware-specific work behind the appropriate IO interface.

    Working Principles

    • Keep hardware details inside IO implementations.
    • Keep subsystem logic deterministic and based on logged inputs.
    • Prefer commands for behavior sequencing and subsystems for mechanism state/control.
    • Use AdvantageScope and logs to debug behavior before changing constants blindly.
    • Treat simulation and replay as first-class workflows, not afterthoughts.
    • Update documentation when adding a subsystem, controller binding, autonomous routine, dashboard output, or required setup step.
