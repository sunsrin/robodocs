    103: Controls & Operations¶

    This page summarizes the current operator interface and the robot behaviors wired in RobotContainer. Treat it as a software reference for drive practice, simulation, and code review. If controls change in configureButtonBindings(), update this page in the same pull request.

    Controller Ports¶

    Device Port Code object
    Primary driver controller 0 RazerWolverineController primary
    Secondary operator controller 1 CommandXboxController secondary
    Override switches 5 OverrideSwitches overrides

    The robot creates alerts for missing primary, secondary, and override controllers. Primary and secondary disconnects are warnings; override disconnect is informational.

    Driving¶

    The default drive command is DriveCommands.joystickDrive. It combines primary and secondary joystick inputs:

    • Forward/backward: -leftY.
    • Left/right: -leftX.
    • Rotation: -rightX.
    • Driver and secondary values are added together before deadband and scaling.

    By default the drive is field-relative. Override switch 1 changes to robot-relative drive.

    The joystick processing applies a 0.1 deadband and squares magnitude for finer low-speed control. Linear output is scaled by DriveConstants.maxLinearSpeed; angular output is scaled by DriveConstants.maxAngularSpeed.

    Primary Controller Bindings¶

    Input Behavior
    Left bumper Align while launching, spin flywheel to tracked target, move hood to tracked target, and feed through hopper/kicker when all launch tolerances are met.
    Right bumper Force launch when hub state allows it, bypassing normal tolerance checks. Secondary right bumper can also trigger this.
    B Outtake through hopper, kicker, and slamtake roller.
    Right claw Unjam by outtaking hopper and kicker.
    Lower right paddle Outpost preset for flywheel and hood.
    Left claw Passing preset for flywheel and hood.
    Lower left paddle Tower preset for flywheel and hood.
    Upper left paddle Trench preset for flywheel and hood.
    Upper right paddle Force trash compactor to max position while held.
    Right trigger Retract intake and run intake briefly. Secondary right trigger can also trigger this.
    Left trigger Run intake and deploy slamtake while held in teleop. Secondary left trigger can also trigger this.
    A Drive under tower with a locked tower-facing rotation.
    X Force trash compactor down while held.

    The left-bumper launch flow also rumbles the secondary controller if the robot is ready to launch but the hub period is inactive and the robot is not in passing mode.

    Secondary Controller Bindings¶

    Input Behavior
    Start + Back Reset gyro heading while preserving current translation.
    A Systems-check preset at minimum hood angle.
    Y Systems-check preset at maximum hood angle.
    X Run hood zero, slamtake home, and trash compactor min home unless B is also held.
    X + B Home trash compactor to max.
    Double-press B Force hood zero, force slamtake max zero, and force trash compactor min zero.
    POV right Increase hood angle offset by 0.2 degrees, repeating while held.
    POV left Decrease hood angle offset by 0.2 degrees, repeating while held.
    POV up Force trash compactor to max while held.
    POV down Force trash compactor to min while held.
    Left bumper Test flywheel spin-up at 200 rad/s while held.
    Right bumper Force launch, same as primary right bumper.
    Left trigger Run intake.
    Right trigger Retract intake and run intake briefly.

    Override Switches¶

    Switch Behavior
    Driver switch 1 Robot-relative drive override.
    Driver switch 2 Superstructure coast override when disabled.
    Multi-direction left Force hub shift result to lost auto.
    Multi-direction right Force hub shift result to won auto.
    Operator switch 0 Disable flywheel auto spin-up.
    Operator switch 1 Ignore hub state and allow external hub counter override behavior.
    Operator switch 2 Ignore tilt checks for launch tolerance.

    Coast override is only enabled while disabled. It is cleared automatically when the robot is enabled.

    Automatic Mode Behavior¶

    When teleop starts:

    • The slamtake deploys.
    • Hood, slamtake, and trash compactor home if they are not already zeroed.
    • Hub shift timing is initialized.

    When autonomous starts:

    • Hub shift timing and hub counter are initialized.
    • Object detection fuel poses are initialized.
    • Hood, slamtake, and trash compactor are force-zeroed.
    • The slamtake deploys after a short delay.
    • Hood runs its autonomous command.
    • Intake and flywheel automation run after configured delays.
    • The trash compactor changes behavior based on neutral-zone trench bounds.

    When disabled:

    • Hub shift timing is reset.
    • Coast override is cleared on the next enable edge.

    Dashboard Outputs¶

    RobotContainer.updateDashboardOutputs() publishes key driver information:

    Dashboard key Meaning
    Drive At Goal Whether drive yaw is within launch/pass tolerance.
    Drive Pitch & Roll OK Whether tilt is within tolerance for launching or passing.
    Match Time Current Driver Station match time.
    Shifts/Remaining Shift Time Remaining time in the current hub shift period.
    Shifts/Shift Active Whether the shifted hub period is active.
    Shifts/Game State Current shifted hub state.
    Shifts/Active First? Whether the alliance is active first.
    Hood Angle Offset Current hood offset shown with one decimal place.

    The AprilTag layout chooser also creates an alert if a non-default layout is selected.
